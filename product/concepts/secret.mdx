A **secret** is a **named** credential stored in your workspace: typically an API key or token. You use secrets when you set up **[LLM providers](/product/concepts/llm-provider)** and **HTTP integrations** (for example in headers or `{{secrets.YOUR_NAME}}` templates).

## Adding a secret

In **Secret Manager**, click **Add Secret** (or use **Quick Create → New Secret**). You choose:

- **Name** — unique in the workspace; use something you will recognize (for example `openai-production`).
- **Kind** — **User** or **System** for your own organization / policy.
- **Value** — pasted once; after you submit, the dashboard **does not show the value again**.

Keep a copy somewhere safe outside Phrony if your compliance rules require it.

## Rotating a secret

There is **no “edit value”** screen. To rotate:

1. **Add Secret** with a new name (or reuse after deleting the old one, if unused).
2. Open **LLM Providers** or **Integrations** and **point** them at the new secret.
3. **Remove** the old secret from **Secret Manager** if nothing references it anymore.

## Names must be unique

If the name is already taken, **Add Secret** fails with a clear message—pick another name.

## If a run cannot read a secret

When a secret is missing, renamed, or corrupted, runs may fail with a credential or decryption message. Fix the binding in **LLM Providers** or integration settings, or recreate the secret.
