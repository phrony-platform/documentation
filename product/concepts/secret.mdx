## 1. What is a secret

A **secret** is a **named** credential stored in the workspace—usually an API key or token. **[LLM providers](/product/concepts/llm-provider)** and **HTTP [integrations](/product/concepts/integration)** reference secrets so configuration can say *which credential* without ever re-displaying the raw value in the UI.

### 1.1 Field reference

| Field (dashboard) | What it is | Rules and limitations |
| --- | --- | --- |
| **Name** | Identifier in pickers and templates (e.g. `{{secrets.NAME}}`). | **Unique** per workspace. |
| **Kind** | **User** vs **System** (or equivalent). | Organizational/policy label per your setup. |
| **Value** | The secret material. | Entered **once** at creation; dashboard **does not show** it again afterward. |

There is **no** “edit value” screen—rotation means **new secret** + rebind consumers.

## 2. How to create a secret

1. Open **Secret Manager** as **Owner** or **Admin** (see **Permissions**).
2. **Add Secret** (or **Quick Create → New Secret**).
3. Enter **name**, **kind**, **value**; submit.

## 3. How to update a secret

Values cannot be updated in place. **Add** a new secret with a new name (or after removing the old one if unused), then **edit** each **[LLM provider](/product/concepts/llm-provider)** or **HTTP integration** to select the new secret, then **remove** the old secret if nothing references it.

## 4. How to delete a secret

In **Secret Manager**, **remove** the secret only after **no provider or integration** still references it; otherwise runs fail until bindings are fixed.

## 5. Permissions

| Action | Owner | Admin | Developer | Operator |
| --- |:---:|:---:|:---:|:---:|
| View secret **names** (Secret Manager) | Yes | Yes | Yes | **No** |
| **Add** / **remove** secrets | Yes | Yes | **No** | **No** |

## 6. Limitations

- Name uniqueness enforced on create.
- Historical runs do not “rewrite” if you rotate—new sessions pick up new bindings.

## 7. Notes

- Keep a copy outside Phrony if compliance requires—you cannot view the value back from the UI.

## 8. Cautions

- Deleting a secret that is still selected on a provider or connector causes **immediate** auth failures.

## 9. Common errors

| Symptom | Typical cause |
| --- | --- |
| Add fails | **Name** already taken. |
| Runs fail / decrypt errors | Secret missing, renamed, or wrong binding. |
