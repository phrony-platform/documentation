## 1. What is an LLM provider

An **LLM provider** is a **named connection** in your workspace that tells Phrony how to reach models: **OpenAI**, **Azure OpenAI**, **Anthropic**, or **Self-hosted** (OpenAI-compatible). Every **[agent](/product/concepts/agent)** references exactly **one** provider so runs know **where** to send completions.

### 1.1 Field reference

| Field (dashboard) | What it is | Rules and limitations |
| --- | --- | --- |
| **Name** | Label in lists and pickers. | **Unique** in the workspace. |
| **Provider type** | Vendor / mode. | Drives defaults and required fields (**Self-hosted** always needs **Base URL**). |
| **API key secret** | **[Secret](/product/concepts/secret)** holding the key. | Raw value **never** shown after secret creation; pick from **Secret Manager**. |
| **Base URL** | HTTP root for API calls. | **Required** for **Self-hosted**; optional override for other types (proxy/gateway). |

## 2. How to create an LLM provider

1. Open **LLM Providers** (builders with access—see **Permissions**).
2. **Owner** or **Admin** only: **New** / **Create** (or **Quick Create → New LLM Provider**).
3. Fill **name**, **type**, **secret**, **base URL** as required; save.

**Developer** can list and open providers but **cannot** create or edit them.

## 3. How to update an LLM provider

**Owner** or **Admin** opens the provider → **Edit** → change name, type, secret, or URL → **Save**. Validation re-runs (especially **base URL** for self-hosted).

## 4. How to delete an LLM provider

If your build exposes **delete**, use it only when **no agents** still reference the provider; otherwise runs fail until agents are repointed. Prefer **edit** to a new secret or retire agents first.

## 5. Permissions

| Action | Owner | Admin | Developer | Operator |
| --- |:---:|:---:|:---:|:---:|
| View **LLM Providers** | Yes | Yes | Yes | **No** |
| Create / edit / delete provider | Yes | Yes | **No** | **No** |

## 6. Limitations

- Workspace **plan** may cap provider **count**—see **[Limits & plans](/product/reference/platform-rules)**.
- Invalid or unreachable **base URL** blocks saves or runs depending on validation.

## 7. Notes

- Rotating keys means **new [secret](/product/concepts/secret)** + **edit provider** to point at it.
- Agents cache the provider **by reference**; changing the provider affects the next runs.

## 8. Cautions

- Deleting a **secret** still bound to a provider breaks runs until you fix the binding.

## 9. Common errors

| Symptom | Typical cause |
| --- | --- |
| **Save** fails | Missing **base URL** (self-hosted), bad URL format, or missing secret selection. |
| Runs fail with auth errors | Revoked API key, wrong secret, or wrong endpoint. |
| Duplicate name | Pick another **name**. |
