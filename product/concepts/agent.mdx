## 1. What is an agent

An **agent** is the top-level unit you configure in Phrony: a named worker tied to one **[LLM provider](/product/concepts/llm-provider)** and one or more **[agent versions](/product/concepts/agent-version)** (snapshots of prompts, allowed **operations**, and limits). You need agents so the platform knows **which model connection to use**, **how the agent may be started** (directly or only as a sub-agent), and **which version line is live** for automation.

Agents do **not** hold the detailed behavior—that lives on **versions**. The agent row answers: *what is it called*, *Request / HITL / Sub-agent*, and *which provider*.

### 1.1 Field reference

| Field (dashboard) | What it is | Rules and limitations |
| --- | --- | --- |
| **Name** | Label for lists, triggers, and sessions. | Must be **unique** in the workspace. Duplicate names are rejected on save. |
| **Execution mode** | **Request**, **HITL**, or **Sub-agent**. | **Not** the same as **Sequential / Parallel** on **[Agent version](/product/concepts/agent-version)** → Multi-agent (that only governs sub-agent *calls*). **Sub-agent** hides the **Triggers** tab; only parent agents invoke sub-agents as delegated **operations**. Switching **to** Sub-agent checks your plan’s **sub-agent agent** cap. |
| **LLM provider** | Workspace connection used for all versions of this agent. | Exactly **one** per agent. Required at creation; change via **Edit agent**. If the provider becomes invalid, runs fail until fixed. |

## 2. How to create an agent

1. Open **Agents** in the sidebar (**Owner**, **Admin**, **Developer**).
2. Choose **New Agent**.
3. Set **name**, **execution mode**, and **LLM provider**, then save.

**Quick Create → New Agent** opens the same flow with **New Agent** ready.

![New Agent dialog — name, execution mode, LLM provider](/images/create-agent.png)

## 3. How to update an agent

1. In **Agents**, use **Edit agent** on the row (or open the workspace and edit from the header).
2. Change **name**, **mode**, or **LLM provider** as allowed; save.

Changing the provider affects **future** runs; existing session data still shows the version that ran.

## 4. How to delete an agent

1. In **Agents**, open the row’s overflow (**…**) and choose **Delete**.
2. Confirm only if **no version is live** (deployed). If a version is still live, the UI explains that you must deploy elsewhere or roll back before delete is allowed.

## 5. Permissions

| Action | Owner | Admin | Developer | Operator |
| --- |:---:|:---:|:---:|:---:|
| View **Agents** list and open workspace | Yes | Yes | Yes | **No** |
| Create agent | Yes | Yes | Yes | **No** |
| Edit agent (name, mode, provider) | Yes | Yes | Yes | **No** |
| Delete agent (when allowed) | Yes | Yes | Yes | **No** |

Operators use **[Approvals](/product/dashboard/approvals)** and do not see the **Agents** section in the nav.

## 6. Limitations

- **Plan** may cap how many **Sub-agent** mode agents you can have; at the cap, switching mode shows an upgrade or capacity message.
- **[Guard & anomaly](/product/dashboard/guard-anomaly)** can **block** an agent until reviewed; while blocked, **Test run**, **triggers**, and API starts refuse (see **[Limits & plans](/product/reference/platform-rules)**).

## 7. Notes

- **[Agent version](/product/concepts/agent-version)** holds model, instructions, schemas, operations, session caps, and multi-agent settings.
- **Sub-agent** agents still have their own provider and versions; they are not scheduled or API-started on their own. Conceptual guides: **[Multi-agent](/product/multi-agent/overview)**.

## 8. Cautions

- Do not confuse **execution mode** with **Execution model** (Sequential vs Parallel) on a version—both appear in docs and UI but control different things.
- Deleting an agent is irreversible; ensure no production dependency on its live line first.

## 9. Common errors

| Symptom | Typical cause |
| --- | --- |
| **Save** fails on name | Name already used by another agent in the workspace. |
| **Delete** unavailable or dialog blocks | A version is still **live**; deploy another version or change live line first. |
| Cannot switch to **Sub-agent** | Plan cap on sub-agent agents, or validation error from the server. |
| Runs never start on this agent | **[Guard](/product/dashboard/guard-anomaly)** block, plan limits, or invalid provider/secret—see **[Limits & plans](/product/reference/platform-rules)**. |
