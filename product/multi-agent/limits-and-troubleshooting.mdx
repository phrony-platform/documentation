Most numeric and role limits are summarized globally on **[Limits, permissions, and plans](/product/reference/platform-rules)**. This page collects what most often blocks **multi-agent** work.

## Plan and inventory

- **Multi-agent** feature — if **Version configuration** has no **Multi-agent** tab, the workspace plan or hosting may not include delegation.
- **Sub-agent agents** — switching an agent **to** **Sub-agent** mode can fail at your plan’s **cap** on how many Sub-agent agents you may have.
- **Allowed sub-agents count** — each parent **version** allow-list is capped (UI shows the limit).
- **Delegation depth** — chains of Sub-agents calling Sub-agents cannot exceed the plan’s **maximum depth**; deeper calls fail at run start.
- **Max operation calls** — the **stricter** of **plan** vs **version** wins; the editor **clips** saves to the plan ceiling.

## Configuration mistakes

| Symptom | What to check |
| --- | --- |
| **Multi-agent** tab missing | Plan; parent must be **Request** or **HITL**, not **Sub-agent**. |
| Cannot add a sub-agent | Target not in **Sub-agent** mode; **self-reference**; allow-list **count**; **Can execute sub-agents** off. |
| **Save** fails on parent | Invalid **Anomaly control (L1)** rules; empty Sub-agent **description** on a child you rely on. |
| **Save** fails on child | **Description** missing (**Sub-agent** requirement). |

## Runtime failures

| Symptom | What to check |
| --- | --- |
| Delegation errors / child not found | Child agent has **no live** deployed version; deploy the Sub-agent. |
| Depth exceeded | Too many nested Sub-agent calls for your plan. |
| Session or token cap | **[Agent version](/product/concepts/agent-version)** **Max session tokens** / **duration**; reduce parallel work or raise caps. |
| Wrong child behavior | **Test run** uses **selected** parent snapshot, but **child** always uses **live**—deploy child changes you expect in production paths. |

## Execution model surprises

- **Parallel** only parallelizes multiple calls to the **same** Sub-agent with **different** inputs in **one** turn. Different Sub-agents in the same turn are **serialized**.
- If you see only **one** child run when you expected several, confirm **Parallel** is selected and parent **instructions** ask for **one sub-agent operation call per item**, not one batched payload. **[Sequential and Parallel](/product/multi-agent/sequential-and-parallel)**.
