## Session = one thread of work

A **[session](/product/concepts/session)** starts when a **root** **[run](/product/concepts/run)** begins (for example parent **Test run** or **trigger**). Any **Sub-agent** delegations from that root create **child** runs that **stay in the same session**.

- **Triggered by** and **Trigger type** on the session reflect how the **root** run started.
- **Status** rolls up high-level progress; individual runs have their own statuses.
- **Tokens** on the session **aggregate** root and child usage where applicable.

Session-wide caps (**Max session tokens**, **Max session duration**) on the **parent [agent version](/product/concepts/agent-version)** apply to the **whole** session, including children.

## Root vs child runs

| Run | Role |
| --- | --- |
| **Root** | The parent agent’s run—the one you usually open from **Sessions** or **Test run**. |
| **Child** | A Sub-agent execution triggered by an **operation** call from the parent (possibly several children per parent turn). |

Child runs have their own **tokens**, **trace**, and **agent version** label (the **live** child snapshot that executed). **[Run](/product/concepts/run)** inspector shows **Sub-agent** sections and, for **Parallel** batches, grouping may vary by release.

## Inspector and trace

- **Trace / timeline** follows **Settings → General → Session trace layout** (**Basic** vs **Advanced**).
- **Pause / review** can appear on the root or on a child when **HITL** or **Guard** pauses apply.
- **Terminate session** (from **Dashboard** home) fails **all** in-progress runs in that session—including children.

## Billing and limits

- **Monthly session** counts and plan inventory apply when a **new** session starts.
- **Max operation calls** and per-run token limits apply **per run**; session caps apply **across** root + children.
