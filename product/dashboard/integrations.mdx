Open **Integrations** from the sidebar. The list uses **Add Integration**; the empty state says there are no integrations yet and suggests adding **HTTP** or **built-in** connectors, then defining operations on the integration page.

## Add Integration

The dialog title is **Add Integration**.

Choose **Integration type**:

- **HTTP** — *HTTP connectors call external APIs. Secrets hold credentials; they are never shown here after you pick them.*  
  After filling **Name** and HTTP connection fields, **Create** saves the connector (footer shows **Cancel** and **Create**).

- **Built-in** — *Built-in connectors use managed integrations. Choose a toolkit and sign in or provide an API key.*  
  Saving may complete connection flows; on success you can be taken to that integration to **add an operation**.

**Name** field placeholder: *Label shown in the dashboard and when you assign operations to agents*

Default type in the dialog is **Built-in** (toggle to **HTTP** if needed).

## Delete integration

**Delete integration?** warns:

- *This action cannot be undone.*
- *This removes the integration and all operations. Agent versions that allowed those operations will lose those links.*

## Plan limits

Your subscription may cap **custom connectors** or related resources. At the limit, save or create flows show upgrade or capacity messaging.

## Built-in connectors and hosting

**Built-in** integrations need your team to finish **connection** steps in the dashboard **and** correct **hosting** on the Phrony side. If hosting is incomplete, **Test run** or trigger executions can fail at start even when the agent version saved successfully.

## Event triggers

**Event** triggers need a **built-in** integration with an **active** connection. See **[Creating triggers (wizard)](/product/dashboard/triggers-wizard)** and **[Trigger](/product/concepts/trigger)**.

## Operations and agents

On an agent’s **Version configuration → Operations** tab, you choose which **operations** from these integrations this version may call. Only operations in your workspace appear.
