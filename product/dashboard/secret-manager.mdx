---
title: "Secret Manager"
description: "Add and remove secrets; who can see and change them."
---

1. Open **Secret Manager** from the sidebar (or **Quick Create → New Secret**).
2. Click **Add Secret**. Enter a **unique name**, choose **Kind** (**User** or **System**), paste the **value**, and submit.
3. After saving, the list shows **name**, **kind**, and **created** time—the **value is never shown again** in the table or in an “edit” form.
4. To **remove**, use the row action and confirm. If a secret is still used by a provider or integration, remove or update those bindings first or the system may block deletion depending on checks.

## Who can add or delete

- **Owner** and **Admin** see **Add Secret** and delete actions.
- **Developer** can **view** the list (names and metadata) but **cannot** add or delete—the buttons are not offered.

## Rotating credentials

Because there is no **edit value** screen, **add a new secret**, **switch** **LLM Providers** and **Integrations** to use it, then **delete** the old secret if nothing else needs it.

## If the name is taken

Phrony shows an error when you try to reuse a name—pick a different **Name** and save again.
