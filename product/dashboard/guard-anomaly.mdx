**Guard** in the sidebar opens anomaly and review workflows when your workspace has access; it may show a **Beta** label. If the feature is not enabled, **Plan & billing** or other flows may show a short message that Guard is not enabled for the workspace yet.

## Anomaly control (hub)

The **Anomaly control** page explains that you can review suggested rule changes from automated checks, inspect incidents, and cross-link from blocked agents. It notes that **inline Path-1 rules** are edited on each agent version under **[Agents](/product/dashboard/agents)** (see **Limits & Safety** on **[Agent workspace](/product/dashboard/agent-workspace)**).

Two actions:

- **Rule proposals** — list of proposed rule changes from async detection.
- **Incidents** — list of anomaly incidents.

### Rule proposals

The **Rule proposals** page states that accepting a proposal **creates a new agent version in Testing** with the proposed ruleset. You can **accept** or **reject** proposals from the list; success toasts confirm the outcome.

### Incidents

**Anomaly incidents** lists incidents; you can open a detail view per incident. Navigation links return to the hub.

## Approvals and anomalies

Pending anomaly-related user tasks also appear under **[Approvals](/product/dashboard/approvals)** when you filter by **Anomaly**.
