Open **Settings** from the bottom of the sidebar. The page subtitle is **Workspace preferences and integrations.**

## Tabs

### General

- **Appearance** — **Light**, **Dark**, or **System** (follow device or browser). Copy: *Select how the dashboard should look. You can change this at any time.*
- **Session trace layout** — how steps appear next to the run tree in **[Run inspector](/product/dashboard/run-inspector)**:
  - **Basic mode**
  - **Advanced mode**

### Workspace

Members, invitations, and other workspace-level controls (wording and fields depend on your Phrony version and role). Builders with access manage who belongs to the workspace here.

### Plan & billing

Visible to roles that can manage billing (for example **Owner** / **Admin** per your workspace).

The panel includes:

- **Current plan** — plan name and status; actions such as **Upgrade to Premium**, **Upgrade to Business** / **Update seats**, and **Manage subscription** (opens the billing portal).
- **Usage this period** — dated usage for the billing period.
- **Add-ons** — for example packs that add integration call allowance; dialogs explain stacking and that the portal shows exact charges.

A dialog for **Business** seats explains billed seats vs occupied seats (members and pending invites) and estimated monthly totals.

If **Guard** (anomaly features) is not enabled for the workspace, a short **Guard** dialog may state that the feature is not enabled yet.

### API keys

Create and manage keys for **automation outside the browser**. This view is embedded in Settings; it is the same capability described from **[Dashboard overview](/product/dashboard/overview)** under API keys.

Deep link: opening **Plan & billing** from notifications uses the billing tab (for example after a plan compliance warning).
