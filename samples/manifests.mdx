After apply, finish **integration connection** in the Phrony dashboard when prompted.

## Telegram + public API sample {#telegram-public-api-sample}

The **`telegram-public-api-notify/`** folder is a **relay demo**: an **API trigger** starts a run on a **HITL** root agent. Each run **fetches one cat fact** from a **read-only public HTTP** API (`https://catfact.ninja/fact`) and **sends that fact to Telegram**.

**Why HITL:** the agent does **not** need to interview the operator about message text—the content is the fact (optionally with one short line of context from the trigger). **Human-in-the-loop** here means the Telegram **send** operation uses **require approval**, so a human can **approve or reject** the outbound message in the Phrony dashboard before it is delivered.

**How to use it**

1. Apply the manifest and set **`telegram_chat_id`** to the destination chat id.
2. Connect the **Telegram** integration (bot token). The **Cat Facts (public HTTP)** integration is declared in the manifest (read-only, no auth).
3. Use the agent’s **API trigger** from your automation (optional context in the payload). The agent fetches a fact and proposes the Telegram send; the operator **approves the send** in the dashboard when prompted.
4. See [Human in the loop (HITL)](/techniques/human-in-the-loop) for execution mode and approvals.

See [Manifest](/concepts/manifest) for document shape, apply, and dry run.
