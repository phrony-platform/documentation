title: "Introduction"
description: "Phrony — an execution environment for governed, agentic systems."
---

Phrony is an **execution environment for agentic systems**. It is the runtime in which AI agents operate — where perception, reasoning, and action are not only possible, but governed.

Phrony is **not** an interface for interacting with a language model. It is **not** a workflow engine. It does not model predefined sequences of steps, nor does it aim to encode deterministic logic. It operates at a different layer of the system: where autonomous decision-making meets real-world interaction.

> Phrony defines not what an agent should do, but **how an agent is allowed to operate**.

This includes:

- Controlling how external **operations** (integrations) are accessed
- Recording how decisions are made and enabling inspection or reproduction of executions
- Introducing structure around inherently dynamic processes without forcing them into fixed flows
- Providing guardrails around agent behavior — enforcing boundaries on what can be accessed, how actions are executed, and how data is handled
