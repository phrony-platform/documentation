## `phrony init`

Creates:

- `manifests/index.yaml` ‚Äî manifest **index** that includes other YAML files
- `manifests/example.yaml` ‚Äî starter `phrony.manifest` document
- `phrony.config.json` ‚Äî workspace hints (`tenantId`, `apiBase`, `rootAgentId`, `defaultProfile`)
- `phrony.config.ts` ‚Äî optional duplicate for editors (the CLI reads **JSON** only)

If `.gitignore` exists (or is created), the CLI appends `.env` and `.phrony/` when missing.

| Flag | Description |
| ---- | ----------- |
| `--force` | Overwrite starter files if they already exist. |

---

## `phrony lint [path]`

Validates manifest YAML against the published schema (with **includes** resolved from disk). Default path: `./manifests`.

- Pass a **file** or a **directory**. Directories are scanned recursively for `.yaml` / `.yml` (skipping `node_modules` and `.git`).
- **Offline** ‚Äî does not call the network; safe in CI for schema checks.
