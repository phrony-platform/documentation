Manifest **lint** in this section is the **CLI** command: it validates YAML **offline** against the published schema. If you work mostly in the **Phrony dashboard**, **Import manifest** and **Apply manifest** run **checks** (placeholders, LLM provider names, and related blockers) in the browser before you **dry run** or **apply**—see [Manifest — Where you use manifests in Phrony](/concepts/manifest#where-you-use-manifests-in-phrony). You can use **`phrony lint`** in CI or before upload for the same declared inputs shape as **`--values`** on **`plan`** / **`apply`**.

## `phrony init`

Creates:

- `manifests/index.yaml` — manifest **index** that includes other YAML files
- `manifests/example.yaml` — starter `phrony.manifest` document
- `phrony.config.json` — workspace hints (`tenantId`, `apiBase`, `rootAgentId`, `defaultProfile`)

If `.gitignore` exists (or is created), the CLI appends `.env` and `.phrony/` when missing.

| Flag | Description |
| ---- | ----------- |
| `--force` | Overwrite starter files if they already exist. |

**In the Phrony dashboard:** you do not need **`phrony init`** to use manifests—create agents and integrations in the UI, **export manifest** when you want YAML on disk, or **import manifest** / **apply manifest** to bring YAML into the workspace. **`phrony init`** is for a **local** starter layout (`manifests/`, `phrony.config.json`) when you prefer files and the CLI.

---

## `phrony lint [path]`

Validates manifest YAML against the published schema (with **includes** resolved from disk). Default path: `./manifests`.

- Pass a **file** or a **directory**. Directories are scanned recursively for `.yaml` / `.yml` (skipping `node_modules`, `.git`, and **`phrony.values.yaml`** / **`phrony.values.yml`**, which are values files rather than manifest documents).
- **Offline** — does not call the network; safe in CI for schema checks.

| Flag | Description |
| ---- | ----------- |
| `--values <path>` | Merge `inputs` from a values file before validation (same behavior as **`phrony plan`** / **`phrony apply`**). |

**In the Phrony dashboard:** parameterized manifests are validated when you map **inputs** in the **Import manifest** or **Apply manifest** dialog; use **`phrony lint --values`** when you want the same merged document checked **before** upload.

For **`plan`**, **`apply`**, and **`diff`**, see [Manifest](/packages/cli/manifest).
