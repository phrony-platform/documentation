## `phrony profile`

Inspect and update **`~/.phrony/credentials`** profiles (see [Credentials](/packages/cli/credentials-and-config#credentials)).

### `phrony profile ls`

Lists each profile in **`~/.phrony/credentials`**. For every profile you see **yes** or **no** for:

- A usable **OAuth** session (refreshable)
- A non-empty stored **`access_token`** (it may still be expired if refresh failed)

Secrets are never printed. The row marked **`→`** is the profile the CLI selects when **`--profile`** is omitted. Resolution order: **`--profile`** → **`defaultProfile`** in **`phrony.config.json`** → the credentials file **`default`** pointer → the profile name **`default`**.

### `phrony profile logout`

Same behavior as [`phrony logout`](#phrony-logout): removes OAuth fields for the selected profile. JSON output uses **`"command": "profile logout"`**.

---

## `phrony logout`

Removes **OAuth** fields for the selected profile from your credentials file (access and refresh tokens, tenant id stored with the session, optional **`api_base`** / **`user_email`**). Other profile keys (for example **`api_key`**) are **not** deleted. If nothing remains in that profile after stripping OAuth, the profile table is removed; if the file’s **`default`** pointer aimed at that profile, it is cleared too.

Same as **`phrony profile logout`**; use whichever fits your mental model.

This only changes your **local** token file. It does **not** revoke refresh tokens on the server (use the Phrony dashboard to revoke access if a machine was compromised).
