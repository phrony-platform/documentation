---
title: "Agent in the loop (AITL)"
description: "How a parent agent and delegation interact during execution — coordination without replacing human oversight (HITL)."
---

**Agent in the loop (AITL)** describes the coordination path where a **parent** [agent](/concepts/agent) invokes **sub-agents** as part of a single run’s reasoning — for example to specialize work or to call out to a governed tool surface that lives in another agent configuration. The full model (sub-agent mode, run tree, step types, and safety) is in [Multi-agent systems](/techniques/multi-agent-systems). AITL is the multi-agent complement to the **human** path ([Human in the loop](/techniques/human-in-the-loop)); both can appear in a complex [session](/concepts/session) depending on your version settings and [integration](/concepts/integration) design.

The parent run stays responsible for orchestration, while sub-agent calls execute under their own version rules and limits, with the platform stitching together traces for [debugging and observability](/when-to-use-phrony#debugging).
